## freeze 冻结，不可增删改

### 冻结的作用

1. 全局常量配置对象（路由、枚举、常量字典），防止业务代码误修改；
2. Vue/React 全局静态数据，避免意外副作用；
3. 第三方传入数据做只读保护

### freeze 能冻结类实例、Map/Set 吗

1. 不可变数据方案对比：Object.freeze/ Immer / Immutable  
2. Vue3 响应式：freeze 后的对象不会被 proxy 代理，减少响应式开销（性能优化点）  
3. 为什么常量配置用 Object.freeze 能优化 Vue 性能？  
答：冻结对象标记不可变，Vue 跳过响应式劫持，节省内存与监听依赖。  
4. freeze 循环引用处理：手写深冻结必须加缓存（WeakMap）防止栈溢出

3. freeze 循环引用处理：手写深冻结必须加缓存（WeakMap）防止栈溢出  
4. 包装对象（new String /new Number）冻结的特殊表现  


```js
const _objectFreeze = object => {
    // 补全代码
    if(typeof object !== 'object' || object === null) {
        throw new TypeError(`the ${object} is not a object`)
    }

    const keys = Object.getOwnPropertyNames(object)
    const symbols = Object.getOwnPropertySymbols(object)

    [...keys, ...symbols].forEach(key => {
        Object.defineProperty(object, key, {
            configurable: false,
            writable: false,
        })
    })

    Object.preventExtensions(object)
}
```

## seal 密封 不可增删

```js
function _objectSeal(object) {
  if (typeof object !== 'object' || object !== null) {
    throw new TypeError(`the ${object} is not a object`);
  }
  const keys = Object.getOwnPropertyNames(object);
  const symbols = Object.getOwnPropertySymbols(object);
  [...keys, ...symbols].forEach((key) => {
    Object.defineProperty(object, key, {
      configurable: false,
    });
  });
  Object.preventExtensions(object);
}
```
